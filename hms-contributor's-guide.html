Contributor's Guide: Git Feature Branch Workflow for hotel-management-system

Section 1: Clone the project, create a feature/hotfix branch, and push changes

  1. Clone the repository: 
    git clone https://github.com/Software-Lifecycle-Consultants/hotel-management-system.git
  2. Go into the project folder: 
    cd hotel-management-system
  3. Create a feature branch (replace "feature-ui-framework" with a relevant branch name): 
    git branch feature-ui-framework
  4. Check which branch you are in: 
    git branch
  5. Move to your feature branch: 
    git checkout feature-ui-framework
  6. Make your changes:

    Add files and edit codes as required.

  7. Check the changes: 
    git status
  8. Add the changes you want to commit: 
    git add .
  9. Commit all the changes with a comment: 
    git commit -m "Comment about the commit"
  10. Push changes to the feature branch:

    To push the main repository, you first have to add the remote server to Git by running:

    git remote add origin https://github.com/Software-Lifecycle-Consultants/hotel-management-system.git

    To finally push the repo, run:

    git push -u origin feature-ui-framework

Section 2: Create a pull request and merge to the main branch

  1. Create a pull request from the web:

    1. Go to the GitHub repository (https://github.com/Software-Lifecycle-Consultants/hotel-management-system.git).

    2. Create a pull request from your feature branch (feature-ui-framework) to the main branch.

    3. Compare the changes between the two branches.

    4. Click on "Create Pull Request."

  2. Review and merge the changes:

    Once the pull request is created, it will be reviewed by the prject leads in the team. If everything is good and the changes are approved, the pull request will be merged into the main branch.
